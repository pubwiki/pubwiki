# Version Control

Pub.Wiki includes a powerful version control system inspired by Git.

## How It Works

Every change you make is automatically tracked. The system uses content-addressable storage, meaning each version is uniquely identified by its content hash.

## Features

### Automatic Saving

Your work is saved automatically as you edit. No need to manually save.

### Snapshot History

View the complete history of your project:

1. Open the History panel
2. Browse through snapshots
3. Preview any version
4. Restore if needed

### Version Cards

Versions are displayed as stacked cards, showing:

- Timestamp
- Change summary
- Parent version
- Content preview

## Branching

Create branches to experiment without affecting the main version:

1. Click "Create Branch"
2. Make your changes
3. Merge back when ready

## Forking

Fork any public artifact to create your own version:

1. Find an artifact on Hub
2. Click "Fork"
3. The artifact appears in your Studio
4. Make it your own!

## Best Practices

- Write meaningful commit messages
- Create snapshots before major changes
- Use branches for experiments
- Keep your main branch stable
