## 1. 快速开始

**最快的上手路径：**

1. 打开编辑器，在欢迎页面配置你的 AI API（地址 + 密钥 + 模型）
2. 选择一个起点（输入想法 / 上传参考文件 / 导入 SillyTavern 角色卡）
3. 点击「开始构建」，AI 将自动生成完整的游戏世界
4. 审阅生成结果，满意后点击工具栏「启动」按钮
5. 选择游戏类型（推荐 Ink 互动小说），开始游玩

> 整个过程只需要你提供一个想法和一个 AI API，其余交给世界构建器。

## 3. 配置 API

点击工具栏的 **API** 按钮，打开配置面板。

### 简单模式（推荐新手）

只需填写三项：

1. **API 基础地址** — 你的 LLM API 端点（如 `https://openrouter.ai/api/v1`）
2. **API 密钥** — 你的 API Key
3. **模型名称** — 点击「获取模型列表」自动发现可用模型，或手动输入

填好后点击「测试连接」，看到延迟数字即表示成功。

### 高级模式

可以为三种用途分别配置不同的模型：

| 模型类型 | 用途 | 推荐 |
|---------|------|------|
| **生成模型** | 创作故事内容 | 需要较好的指令遵循能力，如 Claude、GPT-4o |
| **召回模型** | RAG 检索筛选 | 需要低延迟、快速，如 Gemini Flash |
| **更新模型** | 更新游戏状态 | 需要结构化输出，如 Gemini Flash |

> 提示：三种模型都需要支持结构化输出（Structured Output）。召回和更新模型推荐使用便宜快速的小模型以降低成本。

## 4. AI 世界构建器

这是最省力的建世界方式。点击工具栏 **AI** 下拉菜单中的「世界构建」，或在欢迎页直接进入。

### 4.1 选择起点

三种方式任选其一：

- **一个想法** — 直接用文字描述你想要的世界（例如：「一个赛博朋克风格的城市，有四个势力争夺控制权」）
- **参考文件** — 上传 Markdown、文本或 JSON 文件作为素材
- **SillyTavern Lorebook** — 导入酒馆角色卡或世界书

### 4.2 配置选项

| 选项 | 说明 |
|------|------|
| **叙事驱动 / 数值驱动** | 叙事适合互动小说，数值适合 RPG |
| **跳过组织生成** | 简单故事可以不需要组织势力 |
| **清空现有数据后生成** | 从头开始，不保留已有内容 |

### 4.3 生成流程

AI 会分 6 个阶段自动生成：

1. **设计稿** — 生成世界总纲（基调、故事线、角色/地域/组织清单）。你可以审阅、修改后再确认
2. **开场故事** — 写出背景故事和开场剧情
3. **世界数据** — 设置游戏时间、属性字段等参数
4. **地域** — 生成地区、地点和路径
5. **组织** — 生成势力和领地
6. **角色** — 分批生成玩家角色和 NPC

每个阶段完成后，你可以：
- **满意，下一步** — 继续生成下一阶段
- **修订** — 输入修改意见，AI 会重新调整
- **暂停** — 保存进度，下次恢复继续

> 全部完成后，编辑器会自动填充所有数据。你可以在各标签页中查看和微调。

## 5. 手动编辑游戏数据

如果你想手工打造世界，或者在 AI 生成后做调整，以下是各标签页的使用说明。

### 5.1 世界实体

这是游戏的全局设定：

- **游戏时间** — 点击「编辑」设置年/月/日/时/分。AI 会根据时间推进故事
- **属性字段** — 定义角色可以拥有的属性（如力量、智力、魅力）。填写字段名、显示名和描述
- **设定文档** — 直接在世界实体上绑定文档。可设置优先级和召回条件
- **导演笔记** — 引导 AI 创作方向的元信息：
  - **阶段目标**：当前剧情的叙事目标（如「让主角发现真相」）
  - **笔记**：创作备忘（如「角色 A 还不知道秘密」）
  - **标记**：布尔开关（如 `secret_revealed = false`），用于追踪关键事件
- **自定义组件** — 为角色定义额外的数据结构（如「魔法体系」「声望值」等）
- **世界日志** — 记录世界级别的重大事件

### 5.2 角色

左侧为角色列表，右侧为详细编辑面板。

**创建角色：**
- 点击「添加玩家」创建主角（一个游戏只能有一个玩家角色）
- 点击「添加 NPC」创建非玩家角色
- 支持复制已有角色

**角色数据包含：**

| 区块 | 说明 |
|------|------|
| 基本信息 | 名称、性别、种族、称号 |
| 外貌 | 身体特征、服装描述 |
| 位置 | 所在地域和地点 |
| 所属组织 | 隶属的势力 |
| 属性 | 基于世界属性字段的数值（如力量 10、智力 8） |
| 背包物品 | 物品名称、数量、描述、详情 |
| 状态效果 | 临时增益/减益（如「中毒」「祝福」） |
| 关系 | 与其他角色的关系（朋友/敌人等）及关系值 |
| 自定义组件 | 使用世界定义的组件模板 |
| 设定文档绑定 | 将设定文档目录绑定到角色，存放详细背景、性格、记忆等 |
| 日志 | 角色个人事件记录 |

> 提示：较长的背景描述建议写在「设定文档」中并绑定到角色，而不是直接写在字段里。设定文档支持 RAG 召回，AI 在需要时会自动检索相关内容。

### 5.3 地域

左侧为地域列表，右侧为详细编辑。

每个地域包含：
- **基础信息** — ID、名称
- **地点列表** — 该地域内的具体位置（如「王城大门」「酒馆」）
- **路径** — 连接不同地点的通道，可设为「已发现/未发现」
- **设定文档绑定** — 地域的详细描述

### 5.4 组织

每个组织包含：
- **基础信息** — ID、名称、描述
- **领地** — 控制的地域和地点
- **组织物品** — 共有资源
- **设定文档绑定**
- **日志**

### 5.5 设定文档

这是游戏的知识库，采用树形文件夹结构：

- **新建文件/文件夹** — 按主题组织你的设定（如「世界观/历史」「角色/主角背景」）
- **文档编辑** — 支持 Markdown 格式，有粗体/斜体/标题/列表工具栏
- **文档设置**：
  - **召回条件** — 告诉 AI 什么时候需要这篇文档（如「当需要描写战斗时」）
  - **静态优先级** — 启用后文档始终被包含在 AI 上下文中，而不是按需召回
  - **禁用** — 临时关闭文档，不参与 AI 检索
- **导出/导入补丁** — 选择性导出部分文档，方便分享

> AI 使用 RAG（检索增强生成）机制。每次生成故事时，会先从设定文档中召回相关内容，再基于这些上下文创作。因此，设定文档写得越详细，AI 创作质量越高。

### 5.6 开局故事

两个文本框：

- **背景故事** — 给玩家看的世界背景介绍（游戏开始前展示）
- **开场剧情** — 游戏开始时的第一段叙事（AI 会从这里接续创作）

### 5.7 游戏百科

供玩家查阅的词条列表。每个词条有标题和内容。玩家在游戏中遇到高亮的名词时可点击查看。

### 5.8 剧情历史

记录每一回合的故事内容。通常由 AI 自动生成，你也可以手动查看、编辑或删除。

> 警告：直接编辑剧情历史的 JSON 可能破坏数据结构，建议编辑前先导出备份。

## 6. 启动游戏

点击工具栏红色的 **启动** 按钮。

### 第一步：选择数据来源

- **编辑器最新数据**（推荐） — 使用你在编辑器中编辑的当前数据
- **游戏内最新数据** — 使用游戏中已有的数据（适合继续上次的游戏）

### 第二步：选择游戏前端

| 游戏类型 | 适合场景 |
|---------|---------|
| **Ink 互动小说** | 文字冒险，选择驱动的分支故事 |
| **自定义游戏（Galgame）** | 视觉小说，带立绘和对话框 |
| **自定义模板** | 开发者用的空白模板 |
| **测试游戏** | 功能测试用 |

选择后游戏会在当前页面加载，编辑器面板仍可通过浮动按钮打开。

## 7. Ink 互动小说玩法

### 基本流程

1. 游戏显示开场背景故事
2. 点击「开始冒险」进入故事
3. AI 生成一段叙事和若干选项
4. 你选择一个选项（或输入自定义行动）
5. AI 根据选择继续推进故事
6. 循环往复

### 界面说明

**右侧主区域**
- 故事正文以打字机效果逐字显示
- 文中的角色名、组织名、地点名会高亮，点击可查看详情
- 每段故事下方显示选项按钮

**左侧面板**（点击左上角按钮切换）
- **世界概览** — 查看所有角色、地域、组织的当前状态
- **故事时间线** — 查看所有存档点，可以回溯到任意节点
- **发布文章** — 将故事发布为文章

**右上角设置**

| 设置 | 说明 |
|------|------|
| 第二人称/第三人称 | 切换叙事视角（「你走进了...」vs「他走进了...」） |
| 骰子系统 | 关/明骰/暗骰 — 为困难选项添加随机判定 |
| 段落数 | 每次生成的段落数量（10-24） |
| 打字速度 | 文字显示速度（20-120 字/秒） |

### 骰子系统

开启后，某些困难选项会触发掷骰判定：

- **明骰模式** — 难度选项会标注难度值，你需要手动掷骰
  - 掷骰结果 ≤ 难度值 = 成功
  - 失败可以「再试一次」或「接受命运」
- **暗骰模式** — 所有行动自动隐藏掷骰，结果直接影响故事

### 导演指令

在选择区域有一个「导演指令...」输入框，可以给 AI 额外的创作指导（如「这一段请侧重描写感情张力」），它不是角色行动，而是影响叙事风格。

### GM 模式

在左侧面板中可以进入 GM（游戏大师）模式。你可以直接输入指令修改游戏世界状态，例如：

- 「给主角增加 500 金币」
- 「将主角传送到王城」
- 「让某 NPC 受伤」

AI 会分析你的指令，展示建议的变更，你批准后执行。

### 特殊操作

在时间线面板中，每个存档点提供：

- **干涉过去** — 回到之前的时间点，丢弃之后的进度
- **重写史书** — 回到之前的时间点，并将之后的回合压缩重写为一段高密度历史
- **重新生成** — 对当前这段故事不满意，重新生成
- **发布存档** — 将存档点发布，让其他玩家从这里开始

## 8. Galgame 视觉小说玩法

### 基本流程

与 Ink 类似，但以视觉小说的形式呈现：

1. 故事以对话框 + 角色立绘的形式展示
2. 每句对话配有角色表情（平静/开心/愤怒/悲伤/惊讶/害羞等）
3. 点击画面推进对话
4. 出现选项时进行选择

### 界面元素

- **场景背景** — 根据当前场景显示
- **角色立绘** — 说话角色的全身像
- **对话框** — 底部对话框显示角色名和台词
- **控制栏** — 自动推进、快进、对话记录、设置

### 角色立绘管理

通过导航菜单中的「角色立绘」进入立绘管理：

- 为每个角色上传全身立绘
- 为每种表情上传对应头像
- 支持导出/导入立绘包

### 图像 API 配置

Galgame 支持可选的 AI 图像生成功能：

- **立绘 API** — 根据角色描述生成全身立绘
- **表情 API** — 基于立绘生成各表情头像（图生图）
- **CG API** — 生成 CG 插画（即将推出）

通过「资源管理器」可以批量生成缺失的立绘和表情。

## 9. 文件与存档

### 导出

点击 **文件 → 导出文件**，选择格式：

- **JSON 文件** — 标准数据格式，体积较小
- **PNG 图片** — 将数据嵌入图片中，方便分享（可以选择封面图）

### 导入

点击 **文件 → 导入文件**，支持导入 JSON 和 PNG 格式。

### 存档管理

在游戏中，系统会自动创建存档点（checkpoint）。你可以：

- 通过存档管理面板查看所有存档
- 加载任意存档恢复游戏状态
- 删除不需要的存档

### 保存到游戏

`Ctrl+S` 可以将编辑器中的改动同步到正在运行的游戏后端。

## 10. 发布应用

### 设置应用类型

通过 **应用** 菜单切换：

- **编辑器模式** — 打开应用时显示编辑器（适合创作阶段）
- **Ink 互动小说** — 打开应用时直接进入游戏（适合发布给玩家）

### 发布

1. 确认游戏数据完整（无严重验证错误）
2. 点击 **应用 → 发布应用**
3. 系统将保存当前状态并发布

## 12. 常见问题

### Q: 测试连接失败怎么办？

检查以下几点：
- API 基础地址是否正确（通常以 `/v1` 结尾）
- API 密钥是否有效
- 模型名称是否正确
- 网络是否能访问对应的 API 端点

### Q: AI 生成的故事质量不好？

- 使用更强的生成模型（如 Claude Sonnet、GPT-4o）
- 丰富设定文档，提供更多世界观背景
- 使用导演笔记引导 AI 的创作方向
- 调整段落数设置

### Q: 召回模型和更新模型该用什么？

推荐使用便宜快速的模型（如 Gemini 2.5 Flash Lite）。它们只需要做简单的筛选和结构化输出，不需要强大的创作能力。

### Q: 编辑器显示验证错误？

常见错误包括：
- 角色/地域/组织缺少名称或 ID
- 存在重复 ID
- 缺少世界数据或游戏时间

根据错误提示修复即可，也可以忽略继续保存（会有确认弹窗）。

### Q: 简明模式和完整模式有什么区别？

- **简明模式** — 隐藏大部分组件面板（属性、技能、道具、装备等），只保留状态效果。适合轻量级故事或初学者
- **完整模式** — 显示所有组件面板，提供完整的状态管理。适合需要复杂游戏机制的 RPG 场景

### Q: 设定文档的「召回条件」和「静态优先级」怎么用？

- **不设置任何条件** — AI 根据当前情境自行判断是否需要此文档（默认行为，大多数情况足够）
- **召回条件** — 告诉 AI 何时该使用此文档（如「当需要描写战斗时」「当角色处于森林地区时」）
- **静态优先级** — 开启后文档始终包含在 AI 上下文中，不经过 RAG 筛选。优先级数值越大排越前（默认 RAG 结果优先级为 50）

### Q: 世界构建器中途出错了怎么办？

- 点击「暂停」保存当前进度，下次打开会提示恢复
- 点击「重试」重新生成当前阶段
- 对生成结果不满意可以点击「修订」，输入修改意见后 AI 会调整
